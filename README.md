# Аналитика пролонгаций: Автоматизация и BI-отчетность

Проект посвящен автоматизации расчета коэффициентов удержания клиентов (пролонгаций) для отдела сопровождения и визуализации ключевых метрик.

## 📊 Результаты работы
* **Интерактивный дашборд:** [ДАШБОРД В DATALENS](https://datalens.ru/hse6dh28j94m1)
* **Автоматизированный отчет:** Сгенерированный `.xlsx` файл с готовыми расчетами по каждому менеджеру.

![Скриншот дашборд](https://github.com/user-attachments/assets/6d46fa6c-cbd5-45e0-8eac-177dca17ce25)

## 🏗 Архитектура решения и пайплайн работы с данными

При проектировании данного отчета я осознанно отказался от подхода «Классического Excel» (когда сырые данные выгружаются на скрытый лист, а дашборд строится на тяжелых формулах `СУММЕСЛИМН` и `ВПР`). 

Учитывая требования к вакансии (Python, автоматизация, работа с BI-системами), я реализовал **современный пайплайн обработки данных**, разделив логику расчетов и визуализацию:

1. **Backend (Вычислительный движок):** Вся тяжелая логика (очистка текстовых артефактов, приведение типов, сдвиги когорт на M+1 и M+2, агрегация) реализована на **Python (Pandas)**. Это гарантирует высокую скорость обработки сотен тысяч строк, исключает ошибки «человеческого фактора» (случайное удаление ячейки) и позволяет легко масштабировать процесс.
2. **Frontend (Визуализация):** Сгенерированный `.xlsx` файл выступает в роли легкого «снимка данных» (Snapshot) и витрины для руководителя. В нем сохранены базовые расчетные формулы деления (для локального what-if анализа коэффициентов), но сами абсолютные значения подтягиваются уже посчитанными.
3. **BI-Ready:** Параллельно с Excel-отчетом скрипт подготавливает «плоскую» таблицу (CSV), идеально подходящую для мгновенной загрузки в DataLens или любую другую BI-систему.

Такой подход позволяет полностью автоматизировать процесс: при поступлении новых данных за следующий месяц скрипт самостоятельно заберет сырые файлы, пересчитает метрики и выдаст готовый, легкий и читабельный отчет по заданному шаблону.
