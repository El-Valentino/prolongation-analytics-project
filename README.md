# Аналитика пролонгаций: Автоматизация и BI-отчетность

Проект посвящен автоматизации расчета коэффициентов удержания клиентов (пролонгаций) для отдела сопровождения и визуализации ключевых метрик.

* **Интерактивный дашборд:** [Открыть дашборд в Yandex DataLens](https://datalens.ru/hse6dh28j94m1)
* **Автоматизированный отчет:** Сгенерированный `.xlsx` файл с расчетами по каждому менеджеру (очищенный от тяжелых формул).

## 🏗 Архитектура решения и пайплайн работы с данными

При проектировании данного отчета я осознанно отказался от подхода «Классического Excel» (когда сырые данные выгружаются на скрытый лист, а дашборд строится на тяжелых формулах `СУММЕСЛИМН` и `ВПР`). 

Учитывая требования к современной аналитике, я реализовал **надежный пайплайн обработки данных**, разделив логику расчетов и визуализацию:

1. **Backend (Вычислительный движок):** Вся тяжелая логика (очистка текстовых артефактов, приведение типов, сдвиги когорт на 1-й и 2-й месяц, агрегация) реализована на **Python (Pandas)**. Это гарантирует высокую скорость обработки данных, исключает ошибки «человеческого фактора» (случайное изменение данных в ячейках) и позволяет легко масштабировать процесс.
2. **Frontend (Визуализация в Excel):** Сгенерированный `.xlsx` файл выступает в роли легкого «снимка данных» (Snapshot) и витрины для руководителя. В нем сохранены базовые расчетные формулы деления (для локального анализа), но абсолютные значения подтягиваются уже агрегированными.
3. **BI-Ready:** Параллельно со скриптом формируется «плоская» витрина данных (CSV), идеально подходящая для мгновенной загрузки в DataLens или любую другую BI-систему.

Такой подход позволяет полностью автоматизировать процесс: при поступлении новых данных скрипт пересчитает метрики за секунды и выдаст готовый, легкий и читабельный отчет без ручного вмешательства.
